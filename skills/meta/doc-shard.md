# Doc Shard

**Trigger:** "shard doc", "dividir documento", "split markdown", ou quando documento excede context window.

---

## Purpose

Divide documentos markdown grandes em arquivos menores organizados com index. Preserva referências cruzadas, gera table of contents no index, e mantém frontmatter consistente.

---

## When to Use

- Document > 50KB
- Document has many independent sections
- Need to load parts selectively
- Context window constraints

---

## Process

```
1. Parse document structure (headings)
2. Identify logical split points
3. Create index file with TOC
4. Create shard files for each section
5. Update cross-references
6. Verify no information lost
```

---

## Output Structure

```
docs/
├── feature/
│   ├── index.md          # Table of contents
│   ├── 01-overview.md    # Section 1
│   ├── 02-requirements.md
│   ├── 03-architecture.md
│   ├── 04-api.md
│   └── 05-testing.md
```

---

## Index File Format

```markdown
# Feature Documentation

## Table of Contents

1. [Overview](./01-overview.md)
   - Purpose
   - Scope
   - Background

2. [Requirements](./02-requirements.md)
   - Functional
   - Non-functional
   - Constraints

3. [Architecture](./03-architecture.md)
   - System design
   - Data flow
   - Components

4. [API Reference](./04-api.md)
   - Endpoints
   - Request/Response
   - Errors

5. [Testing](./05-testing.md)
   - Strategy
   - Test cases
   - Coverage

---

*Generated by Doc Shard. Original: feature.md (125KB)*
```

---

## Shard File Format

```markdown
---
parent: feature/index.md
prev: 01-overview.md
next: 03-architecture.md
---

# Requirements

## Functional Requirements

...

---

[← Overview](./01-overview.md) | [Index](./index.md) | [Architecture →](./03-architecture.md)
```

---

## Cross-Reference Handling

Original:
```markdown
See [Architecture section](#architecture) for details.
```

After sharding:
```markdown
See [Architecture](./03-architecture.md) for details.
```

---

## Commands

```bash
# Shard a large document
doc-shard split docs/feature.md --output docs/feature/

# Merge shards back (for editing)
doc-shard merge docs/feature/ --output docs/feature-full.md

# Validate shard integrity
doc-shard validate docs/feature/
```

---

## Use Cases

### Loading Selectively
```
User: "How does the API work?"
→ Load only: index.md + 04-api.md
→ Skip: overview, requirements, architecture, testing
```

### Partial Updates
```
Only requirements changed?
→ Update: 02-requirements.md
→ Keep: all other shards unchanged
```
