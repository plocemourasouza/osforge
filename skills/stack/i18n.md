**Trigger:** multi-language, locale, translation, internationalization, next-intl

## next-intl Setup (App Router)

### 1. Install
```bash
bun add next-intl
```

### 2. Config
```typescript
// i18n/config.ts
export const locales = ['en', 'pt', 'es'] as const
export type Locale = (typeof locales)[number]
export const defaultLocale: Locale = 'en'
```

### 3. Middleware
```typescript
// middleware.ts
import createMiddleware from 'next-intl/middleware'
import { locales, defaultLocale } from './i18n/config'

export default createMiddleware({
  locales,
  defaultLocale,
  localePrefix: 'as-needed', // /en/about → /about for default
})

export const config = {
  matcher: ['/((?!api|_next|.*\\..*).*)'],
}
```

### 4. Messages
```json
// messages/en.json
{
  "common": {
    "welcome": "Welcome, {name}!",
    "items": "{count, plural, =0 {No items} one {# item} other {# items}}"
  },
  "nav": {
    "home": "Home",
    "about": "About"
  }
}
```

```json
// messages/pt.json
{
  "common": {
    "welcome": "Bem-vindo, {name}!",
    "items": "{count, plural, =0 {Nenhum item} one {# item} other {# itens}}"
  },
  "nav": {
    "home": "Início",
    "about": "Sobre"
  }
}
```

### 5. Provider
```typescript
// app/[locale]/layout.tsx
import { NextIntlClientProvider } from 'next-intl'
import { getMessages } from 'next-intl/server'

export default async function LocaleLayout({
  children,
  params: { locale },
}: {
  children: React.ReactNode
  params: { locale: string }
}) {
  const messages = await getMessages()

  return (
    <html lang={locale}>
      <body>
        <NextIntlClientProvider messages={messages}>
          {children}
        </NextIntlClientProvider>
      </body>
    </html>
  )
}
```

## Usage

### Server Components
```typescript
import { getTranslations } from 'next-intl/server'

export default async function HomePage() {
  const t = await getTranslations('common')

  return (
    <h1>{t('welcome', { name: 'John' })}</h1>
  )
}
```

### Client Components
```typescript
'use client'
import { useTranslations } from 'next-intl'

export function ItemCount({ count }: { count: number }) {
  const t = useTranslations('common')

  return <span>{t('items', { count })}</span>
}
```

## Date/Currency Formatting

```typescript
import { useFormatter } from 'next-intl'

export function ProductPrice({ price, date }: { price: number; date: Date }) {
  const format = useFormatter()

  return (
    <div>
      <p>{format.number(price, { style: 'currency', currency: 'USD' })}</p>
      <p>{format.dateTime(date, { dateStyle: 'medium' })}</p>
      <p>{format.relativeTime(date)}</p>
    </div>
  )
}
```

## ICU MessageFormat

### Plurals
```json
{
  "cart": "{count, plural, =0 {Cart is empty} one {# item in cart} other {# items in cart}}"
}
```

### Select (gender, etc.)
```json
{
  "greeting": "{gender, select, male {Mr.} female {Ms.} other {}} {name}"
}
```

### Nested
```json
{
  "notification": "{count, plural, =0 {No new notifications} one {{gender, select, male {He} female {She} other {They}} sent you a message} other {# new notifications}}"
}
```

## Key Naming Pattern

Use `namespace.context.element`:

```json
{
  "auth.login.title": "Sign In",
  "auth.login.button": "Continue",
  "auth.login.error.invalid": "Invalid credentials",
  "dashboard.header.greeting": "Welcome back",
  "dashboard.stats.revenue": "Total Revenue"
}
```

---

## Best Practices

1. **Never hardcode user-facing strings** — Always use translation keys
2. **Use namespaces** — Organize by feature/page
3. **Include context for translators** — Add comments in message files
4. **Handle plurals properly** — Use ICU format, not string concatenation
5. **Test with long strings** — German/Finnish can be 30% longer than English
6. **RTL support** — Test with Arabic/Hebrew if needed

## Locale-Prefixed Routing

```
/           → default locale (en)
/pt         → Portuguese
/es         → Spanish
/pt/about   → Portuguese about page
```

```typescript
// Navigation with locale
import { Link } from 'next-intl'

<Link href="/about">About</Link>  // Automatically prefixed
```
